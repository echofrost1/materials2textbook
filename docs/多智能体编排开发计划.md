# 多智能体编排开发计划

> 适用分工：同伴继续按照 `material_pipeline_forward_plan.md` 做资料台账、去重、分类、素材大块和切片处理；本计划负责承接这些中间产物，完成知识组织、教材生成、审核修订的多智能体编排。

---

## 1. 分工边界

### 上游数据处理线负责

- 原始资料只读保存。
- 生成资料台账、去重表、有效素材表。
- 建立素材大块和多归属映射。
- 对选中素材做转码、ASR、关键帧、人工时间码。
- 输出 `video_segments.xlsx` 和 `video_segments.jsonl`。

### 多智能体编排线负责

- 把 `video_segments.jsonl` 转换为统一 `EvidenceChunk`。
- 按章节和知识点组织证据。
- 生成 `ChapterPlan`。
- 生成教材草稿。
- 执行事实审核、引用审核、教学审核。
- 输出修订版教材和审核报告。

两条线的接口是：

```text
work_material1/02_working_processing/json/video_segments.jsonl
        ↓
evidence_chunks.jsonl
        ↓
chapter_plan.json
        ↓
textbook_draft.md
        ↓
review_report.json
        ↓
textbook_final.md
```

## 2. MVP 目标

第一版不追求大模型完全自动生成高质量教材，而是先跑通可复核闭环：

```text
视频片段 JSONL
→ 标准证据片段
→ 章节计划
→ 教材草稿
→ 审核报告
→ 修订提示版教材
```

MVP 验收标准：

- 能读取上游 `video_segments.jsonl`。
- 每个片段能转换为 `EvidenceChunk`。
- 能按 `recommended_chapter` 和 `knowledge_point` 生成章节计划。
- 教材草稿中每个知识点都有证据引用。
- 审核报告能指出待人工复核、缺少证据、教学结构不足等问题。
- 输出文件能被人工直接打开检查。

### 3.1 Agent 角色

| Agent | 当前实现 | 后续增强 |
|---|---|---|
| ResourceAnalystAgent | 规则转换 `video_segments` 到 `EvidenceChunk` | 接入 LLM 修正 ASR、摘要和关键词 |
| KnowledgeOrganizerAgent | 按章节和知识点分组 | 接入聚类、先修关系和章节顺序优化 |
| TextbookWriterAgent | 生成带证据引用的 Markdown 草稿 | 接入 LLM 写作、例题和活动生成 |
| EvidenceReviewerAgent | 检查证据缺失、待审核、评分不足 | 接入事实核验和引用覆盖率 |
| PedagogyReviewerAgent | 检查学习目标、活动、知识点数量 | 接入教学目标和难度梯度审查 |
| RevisionAgent | 汇总审核建议到教材末尾 | 接入 LLM 按审核意见重写正文 |

当前工作流还会额外生成：

- `textbook_outline.md/json`：严格由上游素材片段生成的三级教材目录，当前规则为“章 = recommended_chapter，节 = material_block，知识点 = knowledge_point”。
- `workflow_summary.json`：统计来源记录数、进入编排片段数、章节数、知识点数、审核问题数量和片段状态分布。
- `review_report.md`：给人工审核看的 Markdown 版报告，比 JSON 更适合开会和分工。

## 4. 运行方式

使用当前样例数据运行：

```powershell
python scripts/run_agent_workflow.py
```

指定输入输出：

```powershell
python scripts/run_agent_workflow.py `
  --segments work_material1/02_working_processing/json/video_segments.jsonl `
  --output-dir work_material1/05_final_deliverables/agent_workflow `
  --title "钨极氩弧焊数字教材样章"
```

输出：

```text
work_material1/05_final_deliverables/agent_workflow/
├── textbook_outline.md
├── textbook_outline.json
├── evidence_chunks.jsonl
├── chapter_plan.json
├── textbook_draft.md
├── textbook_draft.docx
├── review_report.json
├── review_report.md
├── workflow_summary.json
├── textbook_final.md
└── textbook_final.docx
```

常用运行模式：

```powershell
# 草稿模式：允许待人工复核片段进入教材草稿，但会在审核报告中标注
python scripts/run_agent_workflow.py

# 正式模式：只使用 review_status 包含 approved 的片段
python scripts/run_agent_workflow.py --approved-only

# 控制每个知识点最多引用几个证据片段
python scripts/run_agent_workflow.py --max-chunks-per-knowledge-point 2

# 过滤教学价值评分过低的片段
python scripts/run_agent_workflow.py --min-teaching-value 0.6
```

## 5. 近期开发计划

### 阶段 1：接口稳定

- 固定 `EvidenceChunk`、`ChapterPlan`、`ReviewReport` 数据结构。
- 和上游确认 `video_segments.jsonl` 字段不频繁变动。
- 对 `Pending_Manual_Timecode`、`approved`、`rejected` 等状态建立过滤规则。
- 使用 `scripts/validate_video_segments.py` 校验上游片段字段、时间码、重复 ID、转写状态和证据完整性。

### 阶段 2：样章生成

- 基于 TIG 样例生成一章教材草稿。
- 严格基于上游片段生成三级教材目录，不主动补充素材中没有的章节。
- 每个段落保留 `chunk_id`、原视频、时间码。
- 标出 ASR 质量低或待人工复核的片段。

### 阶段 3：审核闭环

- 事实审核：检查正文结论是否有证据片段支撑。
- 引用审核：检查每个引用能否回到原始素材和时间码。
- 教学审核：检查章节目标、知识点顺序、活动设计。

### 阶段 4：接入真实 LLM

- 已抽象 OpenAI-compatible Provider，可通过 `ECNU_PLUS_API_KEY`、`ECNU_PLUS_BASE_URL`、`ECNU_PLUS_MODEL` 接入 `ecnu-plus`。
- 已支持 `python scripts/run_agent_workflow.py --use-llm` 用 LLM 替换 TextbookWriterAgent 的正文生成。
- 已支持 `RevisionAgent` 在 `--use-llm` 模式下根据审核报告生成修订后的完整 Markdown。
- 再替换 ResourceAnalystAgent 的摘要、关键词和 ASR 纠错。
- 最后替换 EvidenceReviewerAgent 和 PedagogyReviewerAgent。

### 阶段 5：扩展资料类型

- 接入 PDF / PPT / Markdown 片段。
- 统一转为 `EvidenceChunk`。
- 在章节规划时同时使用视频证据和文档证据。

## 6. 和上游协作约定

上游每次交付给编排线时，建议至少提供：

```text
video_segments.jsonl
video_segments.xlsx
first_round_selected_assets.xlsx
transcripts/
keyframes/
```

字段最低要求：

```text
clip_id
source_asset_id
source_video
original_path
start_time
end_time
subject
material_block
material_block_code
knowledge_point
clip_summary
tags
recommended_chapter
usefulness_score
transcript_status
evidence_text
keyframe_paths
review_status
```

如果 `review_status` 不是 `approved`，编排线可以进入草稿，但必须在审核报告和正文中标记为待复核。
