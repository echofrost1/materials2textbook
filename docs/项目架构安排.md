# materials2textbook 项目架构安排

> 目标：把多源资料处理、知识点组织、多智能体教材生成和质量审核整理成一套清晰、可开发、可验收的系统架构。

> 当前实体数据与生成结果默认在 `/ai/data/materials2textbook`；仓库内不再保存 `work_material1/` 大数据目录。

---

## 1. 项目定位

materials2textbook 的核心不是单点工具，而是一条“素材到教材”的工程流水线：

```text
原始教学资料
→ 可信素材索引
→ 知识片段与证据库
→ 章节规划
→ 教材生成
→ 审核修订
→ 数字教材交付
```

系统面向的输入包括课程视频、音频、字幕、PDF、PPT、Markdown、Excel 台账和人工整理资料。系统输出包括教材 Markdown、章节计划、知识点表、证据引用表、素材索引、审核报告，以及可被前端阅读器渲染的结构化电子教材包。

项目的技术主线可以概括为：

- 多源资料解析：把不同格式的资料统一成可处理的结构化记录。
- 知识切片与证据聚合：把资料拆成可引用、可追溯、可评分的知识片段。
- 多智能体编排：把规划、写作、审核、修订拆成稳定流程。
- 人工可复核：保留台账、来源、时间码、页码和审核意见，方便教师介入。

## 2. 架构原则

### 2.1 原始资料只读

原始客户资料不改名、不移动、不删除。系统只生成台账、索引和处理中间产物，通过 `original_path` 回指原文件。

### 2.2 中间产物结构化

Agent 之间不直接传递大段自然语言，而是传递 JSON / JSONL / Excel / Markdown 组成的标准化产物，例如 `EvidenceChunk`、`ChapterPlan`、`ReviewReport`。

### 2.3 先可信，再自动

第一阶段不追求全自动成书。优先跑通“资料登记、切片、证据回溯、样章生成、人工审核”的闭环，再逐步增强自动检索、聚类和多模态能力。

### 2.4 中心化编排

采用“中心化 Workflow Manager + 专项 Agent”的结构。写作前按流程串行推进，审核阶段允许事实审核和教学审核并行。

## 4. 核心工作流

### 4.1 MVP 主流程

```text
指定资料目录和教材主题
        ↓
建立资料台账
        ↓
去重、粗分类、标记待确认资料
        ↓
对选中素材做深处理
        ↓
生成 evidence_chunks.jsonl
        ↓
抽取知识点并生成 chapter_plan.md / chapter_plan.json
        ↓
生成 textbook_draft.md
        ↓
事实审核 + 教学审核
        ↓
生成 textbook_final.md 和 review_report.md
        ↓
组装 digital_book.json 和前端阅读器资源
```

### 4.2 资料进入系统的标准

资料只有满足以下条件，才进入教材生成阶段：

- 有唯一 `asset_id`。
- 有可回溯的 `original_path`。
- 有资料类型、标题、来源、处理状态。
- 如果是切片资料，需要有时间码、片段路径或文本证据。
- 如果是文档资料，需要有页码、标题层级或段落位置。
- 分类状态不能只依赖 `fallback`。

## 5. Agent 分工

| Agent | 职责 | 输入 | 输出 |
|---|---|---|---|
| Workflow Manager | 控制阶段流转、状态记录、失败重试、人工确认点 | 项目配置、阶段产物 | 任务状态、下一步动作 |
| Resource Analyst Agent | 读取资料片段，提取标题、摘要、关键词、适用章节 | 资料台账、片段文本、元数据 | 标准化知识片段 |
| Knowledge Organizer Agent | 聚类知识点，生成章节结构、学习路径和先修关系 | 知识片段集合 | 章节计划、知识点树 |
| Textbook Writer Agent | 按章节生成教材正文、例子、习题、总结 | 章节计划、证据片段 | 教材草稿 |
| Evidence Reviewer Agent | 检查事实、引用、来源和证据覆盖 | 教材草稿、证据库 | 事实审核报告 |
| Pedagogy Reviewer Agent | 检查学习目标、难度梯度和讲解顺序 | 教材草稿、章节目标 | 教学审核报告 |
| Revision Agent | 根据审核意见修订教材并记录改动 | 草稿、审核报告 | 修订版教材 |
| Media Binder Agent | 将视频、关键帧、图片绑定到对应任务和知识点 | 证据片段、章节计划、资源路径 | 媒体资源块 |
| Digital Book Assembler Agent | 组装项目、任务、正文、媒体、练习和证据引用 | 修订版教材、章节计划、媒体资源块 | digital_book.json |

## 8. 质量控制

### 8.1 必做检查

- 每个核心结论是否有证据片段支撑。
- 每个章节是否覆盖对应学习目标。
- 教材段落是否能回溯到原始资料路径、页码或时间码。
- 章节顺序是否符合先概念、后操作、再应用的教学逻辑。
- 习题是否能从正文或证据片段中找到解题依据。
- `fallback` 和 `uncertain` 资料是否经过人工确认后再进入教材。

### 8.2 可量化指标

- 证据覆盖率：有证据支撑的段落占比。
- 引用完整率：引用能回溯到原始资料的比例。
- 知识点覆盖率：章节覆盖目标知识点的比例。
- 审核通过率：Reviewer 未发现严重问题的章节比例。
- 人工修改率：教师最终修改内容占比。

## 9. 开发里程碑

### Milestone 1：资料台账与样例切片

- 建立样例资料台账。
- 对 TIG 样例视频生成转码、音频、转写、关键帧和片段索引。
- 输出可人工复核的 Excel / JSONL。

### Milestone 2：证据库与知识点组织

- 将视频片段、文档段落统一成 `EvidenceChunk`。
- 抽取标题、摘要、关键词、适用章节。
- 生成知识点树和章节计划。

### Milestone 3：教材草稿生成

- 按章节生成 Markdown 教材。
- 段落保留证据引用。
- 支持插入视频片段、图片或资料来源。

### Milestone 4：审核与修订

- 建立事实审核和教学审核模板。
- 输出审核报告。
- 根据审核结果生成修订版教材。

### Milestone 5：工程化封装

- 将脚本整理为 `src/materials2textbook/` 包。
- 增加配置文件、日志、测试和命令行入口。
- 支持 Markdown / HTML / DOCX 多格式导出。

### Milestone 6：电子教材阅读器

- 输出 `digital_book.json`，描述项目、任务、学习目标、正文、视频、图片、练习和证据引用。
- 复制视频、关键帧和图片到 `digital_book/assets/`。
- 提供本地前端阅读器 `index.html`，支持目录跳转、图文视频混排和基础练习展示。
- 每个媒体资源保留 `chunk_id`、原素材路径和时间码，便于审核追溯。

## 10. 一句话汇报

本项目将多源教学资料处理为可追溯的知识片段，并通过中心化多智能体工作流完成章节规划、教材写作、事实审核、教学审核和修订输出，最终生成带证据引用、素材索引、视频资源和前端阅读器的数字教材。
