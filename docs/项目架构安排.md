# materials2textbook 项目架构安排

> 目标：把论文精读中的“多智能体编排生成”思想落到本项目，形成一套可开发、可迭代、可验收的数字教材生成系统架构。

---

## 1. 项目定位

materials2textbook 不应定位成单一的“视频切片工具”或“一键写教材 Prompt”，而应定位为：

> 面向数字教材构建的多源资料解析、知识点组织与多智能体教材生成系统。

系统输入可以是课程视频、PDF、Markdown、网页文本、PPT 截图等教学资料；系统输出是带有章节结构、知识点、证据来源、视频片段、习题和引用信息的数字教材。

参考论文精读文档后，本项目采用的核心路线是：

```text
资料输入
→ 资料解析
→ 知识切片
→ 证据聚合
→ 章节规划
→ 教材写作
→ 质量审核
→ 修订输出
```

多智能体不是为了“堆角色”，而是为了把上述流程拆成可追踪、可复用、可审核的工程阶段。

---

## 2. 对论文方案的取舍

精读文档中提到的完整方案包含 Resource Collector、Parser、Retrieval Builder、Relevance Judge、Knowledge Organizer、Textbook Writer、Evidence Verifier、Pedagogy Reviewer、Revision Agent 等角色。这个设计适合作为长期目标，但对当前项目来说需要收敛。

### 2.1 当前阶段保留的能力

- 多源资料解析：优先支持 Markdown、PDF 文本、课程视频转写结果和 AssemblyAI 自动章节。
- 知识点切片：把资料切成带主题、摘要、时间戳、来源的知识片段。
- 证据对齐：每个生成段落都能追溯到原始资料片段。
- 章节规划：根据知识点聚类生成教材目录。
- 教材生成：输出 Markdown 格式教材，后续再扩展 HTML / DOCX。
- 审核修订：检查事实、引用、教学逻辑和缺失内容。

### 2.2 暂不作为 MVP 的能力

- 全自动大规模网页爬取：先支持用户提供资料或指定 URL，避免陷入爬虫、版权和清洗复杂度。
- 完整多模态 GraphRAG：先做文本证据索引，图谱和多模态向量作为后续增强。
- 复杂去中心化多 Agent 协作：先采用中心化工作流管理，保证流程稳定。
- 完全自动成书发布系统：先输出结构化 Markdown 和中间 JSON，便于人工检查。

## 4. 推荐 Agent 分工

| Agent | 主要职责 | 输入 | 输出 |
|---|---|---|---|
| Workflow Manager | 控制任务状态、阶段流转、失败重试和人工确认节点 | 用户任务、配置、阶段产物 | 任务状态、下一步动作 |
| Resource Analyst Agent | 读取资料切片，判断主题、摘要、关键词、适用章节 | 原始 chunk、metadata | 标准化知识片段 |
| Knowledge Organizer Agent | 聚类知识点，生成章节目录和学习路径 | 知识片段集合 | 章节大纲、知识点树 |
| Textbook Writer Agent | 按章节生成教材正文、例子、习题和总结 | 章节大纲、证据片段 | 教材草稿 |
| Evidence Reviewer Agent | 检查事实是否有证据支撑、引用是否缺失 | 教材草稿、证据库 | 事实审核报告 |
| Pedagogy Reviewer Agent | 检查教学目标、难度梯度、讲解顺序和学生可理解性 | 教材草稿、章节目标 | 教学审核报告 |
| Revision Agent | 根据审核意见修改教材，并保留修改记录 | 教材草稿、审核报告 | 修订版教材 |

建议采用“中心化流程控制 + 局部并行审核”的模式：写作前严格串行，审核阶段可以让 Evidence Reviewer 和 Pedagogy Reviewer 并行运行。

## 5. 工作流设计

### 5.1 MVP 工作流

```text
用户指定资料目录和教材主题
        ↓
解析资料，生成 raw_resources.json
        ↓
切分资料，生成 evidence_chunks.json
        ↓
抽取知识点，生成 knowledge_points.json
        ↓
组织章节，生成 chapter_plan.md / chapter_plan.json
        ↓
生成教材草稿，输出 textbook_draft.md
        ↓
事实与教学双审核，输出 review_report.md
        ↓
修订教材，输出 textbook_final.md
```

### 5.2 视频资料流程

当前已有 AssemblyAI Auto Chapters + FFmpeg 的切片路线，适合作为视频输入的第一版 baseline：

```text
课程视频
→ AssemblyAI 转写与 auto_chapters
→ chapters.json / transcript.json
→ FFmpeg 切出 clips
→ LLM 将章节标题修正为知识点名称
→ 写入 evidence_chunks.json
```

需要注意：AssemblyAI 的 auto_chapters 更接近“话题分章”，不一定等于严格知识点切片。因此后续要增加 Knowledge Point Extractor，对每个章节再次细分和命名。

## 6. 数据结构约定

为了避免 Agent 之间只传自然语言，建议所有关键中间产物都保存为 JSON，同时生成 Markdown 方便人工检查。

### 6.1 ResourceItem

```json
{
  "id": "resource_001",
  "type": "video|pdf|markdown|web|text",
  "title": "资料标题",
  "path": "原始文件路径或 URL",
  "source": "来源说明",
  "metadata": {}
}
```

### 6.2 EvidenceChunk

```json
{
  "id": "chunk_001",
  "resource_id": "resource_001",
  "title": "知识片段标题",
  "content": "片段正文或字幕",
  "summary": "片段摘要",
  "keywords": ["关键词"],
  "start_ms": 0,
  "end_ms": 120000,
  "clip_path": "outputs/.../clips/01_xxx.mp4",
  "page": null,
  "score": {
    "relevance": 0.9,
    "teaching_value": 0.8,
    "confidence": 0.85
  }
}
```

### 6.3 ChapterPlan

```json
{
  "title": "章节标题",
  "learning_goals": ["学习目标"],
  "knowledge_points": ["chunk_001", "chunk_002"],
  "prerequisites": ["前置知识"],
  "activities": ["例题", "讨论题", "实践任务"]
}
```

### 6.4 ReviewReport

```json
{
  "chapter": "章节标题",
  "fact_issues": [],
  "citation_issues": [],
  "pedagogy_issues": [],
  "missing_content": [],
  "revision_suggestions": []
}
```

## 8. 技术选型建议

### 8.1 后端与流程

- Python 作为主语言。
- Pydantic 定义 ResourceItem、EvidenceChunk、ChapterPlan、ReviewReport 等结构。
- LangGraph 或自定义 DAG 执行器管理多阶段工作流。
- SQLite / DuckDB 保存任务状态和结构化元数据。
- FAISS / Chroma 作为第一版向量索引，可后续替换。

### 8.2 多源解析

- Markdown / TXT：直接读取并按标题、段落切片。
- PDF：先用 PyMuPDF 或 marker 提取文本，保留页码。
- 视频：沿用 AssemblyAI 转写与 auto_chapters，再用 FFmpeg 切片。
- 网页：MVP 阶段先支持用户提供正文或简单 URL 抽取，暂不做复杂爬虫。

### 8.3 模型调用

模型接口应抽象成统一 Provider，避免业务代码绑定某个模型：

```text
LLMProvider.generate(prompt, schema)
EmbeddingProvider.embed(texts)
```

这样后续可以切换 OpenAI、国产模型、本地模型或学校部署模型。

## 9. 质量控制

教材生成必须有审核闭环，否则容易变成“看起来像教材”的长文本。

### 9.1 必做检查

- 每个核心结论是否有 EvidenceChunk 支撑。
- 每章是否覆盖对应学习目标。
- 章节顺序是否符合先易后难、先概念后应用。
- 是否存在无来源的事实陈述。
- 视频片段、页码、链接引用是否能回到原始资料。
- 习题是否能从正文中找到解题依据。

### 9.2 可量化指标

- 证据覆盖率：有证据支撑的段落占比。
- 引用完整率：引用能回溯到原始资料的比例。
- 知识点覆盖率：章节覆盖的目标知识点比例。
- 审核通过率：Reviewer 未发现严重问题的章节比例。
- 人工修改率：教师最终修改内容占比。

## 10. 开发里程碑

### Milestone 1：资料解析与切片

- 支持 Markdown / TXT 输入。
- 支持读取 AssemblyAI 输出的 chapters.json 和 transcript.json。
- 输出 evidence_chunks.json。
- 每个 chunk 保留来源、时间戳、页码或路径。

### Milestone 2：知识点组织

- 对 chunks 抽取标题、摘要、关键词。
- 按主题聚类生成知识点树。
- 输出 chapter_plan.json 和 chapter_plan.md。

### Milestone 3：教材草稿生成

- 按章节生成 Markdown 教材。
- 每段保留证据引用标记。
- 支持插入视频片段链接和资料来源。

### Milestone 4：审核与修订

- Evidence Reviewer 检查事实与引用。
- Pedagogy Reviewer 检查教学逻辑。
- Revision Agent 生成修订版 textbook_final.md。

### Milestone 5：多格式输出

- Markdown 转 HTML。
- Markdown 转 DOCX。
- 生成引用表、资源表和审核报告。

## 11. 一句话汇报版本

本项目参考 AutoGen、MetaGPT、Agentic AutoSurvey 等多智能体编排思想，将数字教材生成拆解为资料解析、知识切片、证据聚合、章节规划、教材写作、质量审核和修订输出等可追踪阶段；系统以结构化中间产物连接各 Agent，优先支持课程视频和文本资料，最终生成带证据引用、视频片段和教学活动的数字教材。
