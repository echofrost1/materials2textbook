# 切片工具说明

> 目标：把课程视频、音频和已整理素材切成可审核、可引用、可进入教材生成流程的知识片段。

---

## 1. 工具定位

切片工具不是最终教材生成器，而是 materials2textbook 的前置处理能力。它负责把长视频、短视频、音频、字幕和人工时间码整理成标准化片段，供后续知识点组织和教材写作使用。

在整体架构中，切片工具位于：

```text
原始资料
→ 资料台账
→ 切片工具
→ EvidenceChunk / video_segments
→ 知识点组织
→ 教材生成
```

切片工具的第一版目标是“可信、可复核、可回溯”，不是完全自动识别所有知识点。

---

## 2. 当前已有能力

仓库当前已有脚本：

```text
scripts/run_first_round_tig.py
```

它用于跑通 TIG 焊接样例素材的第一轮处理：

```text
选中 5 个 TIG 相关视频
→ 保持原始文件不变
→ 转换或复制为 MP4
→ 抽取 WAV 音频
→ 尝试 ASR 转写，失败则标记 ASR_PENDING
→ 抽取关键帧
→ 生成便于人工填写时间码的 video_segments.xlsx 和 JSONL
```

当前样例输出位于：

```text
work_material1/
├── 01_manifest_inventory/

│   ├── first_round_selected_assets.xlsx
│   └── video_segments.xlsx

    ├── audio/
    ├── transcripts/
    ├── keyframes/
    └── json/video_segments.jsonl
```

## 3. 第一版处理流程

### 3.1 输入

第一版以人工选中的素材大块为输入，不做全量自动切片。

建议输入包括：

- 原始视频或音频路径。
- 素材所属课程、章节、知识点大类。
- 人工或半自动生成的时间码。
- 可选字幕、转写文本或讲义文本。

### 3.2 处理

```text
读取选中素材
→ 检查原始路径
→ 获取时长
→ 转码为统一 MP4
→ 抽取 16kHz 单声道 WAV
→ 生成或读取转写文本
→ 抽取关键帧
→ 生成片段表
→ 输出 JSONL
```

### 3.3 输出

核心输出有两类：

```text
人工复核文件：
- video_segments.xlsx
- first_round_selected_assets.xlsx

机器读取文件：
- video_segments.jsonl
- transcript txt
- keyframe jpg
- converted mp4
- wav audio
```

`video_segments.xlsx` 用于人工确认切片边界、知识点名称和教材适用位置。`video_segments.jsonl` 用于进入后续证据库。

## 4. 片段数据结构

建议后续统一成 `VideoSegment`，再转换为总架构中的 `EvidenceChunk`。

```json
{
  "segment_id": "TIG001_SEG001",
  "asset_id": "TIG001",
  "material_block": "钨极氩弧焊",
  "chapter": "钨极氩弧焊基本操作",
  "knowledge_point": "钨极氩弧焊基本原理",
  "start_time": "00:00:00",
  "end_time": "00:02:30",
  "start_seconds": 0,
  "end_seconds": 150,
  "transcript": "片段对应转写文本",
  "summary": "片段摘要",
  "keywords": ["TIG", "焊接", "基本原理"],
  "video_path": "/ai/data/materials2textbook/work_material1/02_working_processing/converted_mp4/TIG001_基本原理.mp4",
  "audio_path": "/ai/data/materials2textbook/work_material1/02_working_processing/audio/TIG001_基本原理.wav",
  "keyframes": [
    "/ai/data/materials2textbook/work_material1/02_working_processing/keyframes/TIG001/frame_01_41s.jpg"
  ],
  "review_status": "pending|approved|rejected",
  "notes": ""
}
```

转换为 `EvidenceChunk` 时，重点保留：

- `asset_id`
- `segment_id`
- 标题、摘要、关键词
- 开始/结束时间
- 视频路径、关键帧路径、转写文本
- 人工审核状态

## 5. 与 AssemblyAI Auto Chapters 的关系

如果后续接入 AssemblyAI Auto Chapters，可以作为自动切片 baseline：

```text
课程视频
→ AssemblyAI 转写与 auto_chapters
→ transcript.json / chapters.json
→ FFmpeg 切出 clips
→ LLM 修正章节标题为知识点名称
→ 人工或审核 Agent 校正
→ 写入 video_segments.jsonl / evidence_chunks.jsonl
```

注意：Auto Chapters 更接近“话题分章”，不等于严格知识点切片。它适合生成初稿，不能直接当作最终知识点边界。

## 7. 人工复核规则

切片结果进入教材生成前，至少确认以下内容：

- 时间码是否能准确覆盖一个完整知识点。
- 片段标题是否是知识点名，而不是笼统视频标题。
- 片段是否过长，是否需要二次切分。
- 转写文本是否存在明显错字、漏字或 ASR_PENDING。
- 关键帧是否能代表片段内容。
- 片段是否适合作为教材证据或示例。

建议使用状态字段：

| 状态 | 含义 |
|---|---|
| `pending` | 已生成，待人工审核 |
| `approved` | 可进入教材证据库 |
| `needs_split` | 片段太长，需要细分 |
| `needs_transcript_fix` | 转写质量不足 |
| `rejected` | 不适合进入教材 |

## 8. 后续演进

### 阶段 1：人工时间码优先

先让教师或项目成员在 `video_segments.xlsx` 中确认片段边界和知识点名称，保证样例链路可靠。

### 阶段 2：ASR + 自动章节辅助

接入 ASR 和 Auto Chapters，自动给出候选片段，再由人工确认。

### 阶段 3：知识点级细切

结合转写文本、关键帧、PPT/讲义标题和章节计划，自动把话题分章细化为知识点片段。

### 阶段 4：进入统一证据库

所有视频片段、文档段落、图片说明统一转换为 `EvidenceChunk`，为教材写作和审核提供来源支撑。
